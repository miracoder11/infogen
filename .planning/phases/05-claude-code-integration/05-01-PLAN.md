# Claude Code Skills for Perceptible Delivery

## Objective
Create Claude Code skills that allow users to trigger demo recording and declare demo statements directly from Claude Code.

## Context
- Phase 4 video capture is complete
- Skills are stored in `.claude/skills/` directory
- Each skill has a `SKILL.md` index and `rules/*.md` files

## Tasks

### Task 1: Create skill directory structure
Create `.claude/skills/perceptible-delivery/` with:
- `SKILL.md` - skill index
- `rules/demo-statement.md` - how to declare demo statements
- `rules/video-recording.md` - how to trigger video recording
- `rules/artifact-collection.md` - how to collect verification artifacts

### Task 2: Implement demo statement skill
The demo-statement skill should:
- Parse demo statement format: `ds-{YYYYMMDD}-{sequence}`
- Store statements in `.infogen/statements/` directory
- Track statement status (pending, verified, failed)

### Task 3: Implement video recording skill
The video-recording skill should:
- Integrate with Phase 4 video capture API
- Provide trigger conditions for automatic recording
- Store video metadata alongside recordings

### Task 4: Implement artifact collection skill
The artifact-collection skill should:
- Collect screenshots, logs, test results automatically
- Store in `.infogen/artifacts/` directory
- Link artifacts to demo statements

## Verification
- Skills are loadable by Claude Code
- Demo statements can be declared and stored
- Video recording can be triggered via skill
- Artifacts are collected and linked

## Output
- `.claude/skills/perceptible-delivery/SKILL.md`
- `.claude/skills/perceptible-delivery/rules/*.md`
- `.infogen/` directory structure

## Done Criteria
- [ ] Skill directory exists with all files
- [ ] SKILL.md properly indexed
- [ ] Each rule file has clear examples
- [ ] Directory structure created
