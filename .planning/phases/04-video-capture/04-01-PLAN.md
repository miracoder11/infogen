# Plan 04-01: Playwright Setup and Core Infrastructure

## Objective

Set up Playwright browser automation infrastructure with video recording capabilities.

## Context

@CONTEXT.md - This is the video capture phase for recording browser interactions.

## Success Criteria

- Playwright installed and configured
- Video recording infrastructure working
- Basic recording API available

## Tasks

### Task 1: Install Playwright Dependencies
type: auto

**Action:**
1. Install Playwright and related dependencies
2. Install video processing utilities (ffmpeg-static)
3. Update package.json scripts

**Implementation:**
```bash
npm install playwright @playwright/test ffmpeg-static
npx playwright install chromium
```

**Verification:**
- `npx playwright --version` returns valid version
- Chromium browser installed

**Done when:**
- Dependencies installed
- Playwright can launch a browser

---

### Task 2: Create Video Recording Configuration
type: auto

**Action:**
1. Create `src/video/types.ts` - Type definitions for video recording
2. Create `src/video/config.ts` - Configuration for video recording
3. Define video output settings (resolution, format, codec)

**Implementation:**
- VideoConfig interface with width, height, fps, outputDir
- Default configuration: 1280x720, 30fps, MP4
- Output directory: `./videos/`

**Verification:**
- Types compile without errors
- Default config is valid

**Done when:**
- Types and config files exist and compile

---

### Task 3: Implement Browser Session Manager
type: auto

**Action:**
1. Create `src/video/browser-manager.ts`
2. Implement isolated browser context creation
3. Implement video recording setup per session
4. Implement cleanup on session end

**Implementation:**
```typescript
export class BrowserSessionManager {
  async createSession(config: VideoConfig): Promise<BrowserSession>
  async closeSession(sessionId: string): Promise<void>
  async cleanup(): Promise<void>
}
```

**Verification:**
- Unit tests for session creation/cleanup
- Isolated contexts work correctly

**Done when:**
- Browser sessions can be created and closed
- Video recording is enabled per session

---

### Task 4: Implement Video Recorder
type: auto

**Action:**
1. Create `src/video/recorder.ts`
2. Implement video recording start/stop
3. Implement video file finalization
4. Handle video encoding with FFmpeg

**Implementation:**
```typescript
export class VideoRecorder {
  async startRecording(session: BrowserSession): Promise<void>
  async stopRecording(): Promise<VideoResult>
  async addAnnotation(timestamp: number, text: string): void
}
```

**Verification:**
- Recording produces valid video file
- Video file is playable

**Done when:**
- Videos can be recorded and saved as MP4

---

### Task 5: Create Video Index and Exports
type: auto

**Action:**
1. Create `src/video/index.ts` - Public API exports
2. Export BrowserSessionManager, VideoRecorder, types
3. Add to main index.ts

**Verification:**
- All exports accessible from main package
- TypeScript compiles without errors

**Done when:**
- Video module is fully exported
