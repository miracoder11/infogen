# Plan 04-02: Playwright Automation with Semantic Selectors

## Objective

Implement browser automation with semantic selectors for reliable, maintainable recording scripts.

## Context

@04-01-PLAN.md - Core video infrastructure is in place. Now build automation layer.

## Success Criteria

- Semantic selector API available
- Common browser actions automated
- Scripts are readable and maintainable

## Tasks

### Task 1: Define Semantic Selector Types
type: auto

**Action:**
1. Create `src/video/selectors.ts`
2. Define semantic selector types (byRole, byText, byTestId, etc.)
3. Create selector builder utilities

**Implementation:**
```typescript
export type SemanticSelector =
  | { role: string; name?: string }
  | { text: string; exact?: boolean }
  | { testId: string }
  | { label: string }
  | { placeholder: string };
```

**Done when:**
- Selector types defined and exported

---

### Task 2: Implement Automation Script API
type: auto

**Action:**
1. Create `src/video/automation.ts`
2. Implement script builder with semantic actions
3. Support common actions: navigate, click, type, wait, screenshot

**Implementation:**
```typescript
export class AutomationScript {
  navigate(url: string): this
  click(selector: SemanticSelector): this
  type(selector: SemanticSelector, text: string): this
  wait(ms: number): this
  waitForSelector(selector: SemanticSelector): this
  screenshot(filename: string): this
}
```

**Done when:**
- Script builder compiles and exports

---

### Task 3: Implement Script Executor
type: auto

**Action:**
1. Create `src/video/executor.ts`
2. Execute automation scripts against browser session
3. Record video during execution
4. Handle errors gracefully

**Implementation:**
```typescript
export class ScriptExecutor {
  async execute(script: AutomationScript, session: BrowserSession): Promise<ExecutionResult>
}
```

**Done when:**
- Scripts can be executed with video recording
