    Add Section 3: 验证产物标准 (Verification Artifact Standards)

    **3.1 概述 (Overview):**
    - Purpose of verification artifacts
    - Traceability requirement: Every artifact must link to its Demo Statement
    - Artifact types: screenshots, logs, test results, videos, diagrams

    **3.2 截图规范 (Screenshot Standards):**

    **Format requirements:**
    - Format: PNG (lossless, web-compatible)
    - Width: 1440px (standard desktop viewport)
    - Height: Variable (capture full content or viewport)

    **Naming convention:**
    ```
    {statement_id}-{action}-{sequence}-{timestamp}.png

    Examples:
    ds-20260325-001-login-form-001-20260325T143000Z.png
    ds-20260325-001-dashboard-view-002-20260325T143015Z.png
    ```

    **Naming components:**
    - statement_id: Links to Demo Statement (required)
    - action: Semantic description of what's shown (lowercase, hyphen-separated)
    - sequence: Zero-padded 3-digit number (001, 002, ...)
    - timestamp: ISO 8601 format without colons (20260325T143000Z)

    **Best practices:**
    - Capture at consistent viewport width
    - Include visible state changes
    - Avoid capturing sensitive data

    **3.3 日志格式 (Log Format):**

    **Structured JSON Lines format:**
    - One JSON object per line
    - Required fields: timestamp, level, statement_id, event
    - Optional fields: message, metadata

    **Log levels:**
    - DEBUG: Detailed debugging info
    - INFO: General progress information
    - WARN: Warning conditions
    - ERROR: Error conditions
    - PASS: Verification passed
    - FAIL: Verification failed

    Reference RESEARCH.md Pattern 3 for complete examples.
