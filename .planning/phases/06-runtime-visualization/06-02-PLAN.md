# Plan 06-02: Execution Trajectory Visualization

## Objective

Create step-by-step execution trajectory view that shows the path taken through agent execution, highlighting branches and decision points.

## Context

Building on the data flow diagram, this adds trajectory visualization that shows execution order, timing, and branch paths.

@CONTEXT.md

## Tasks

### Task 1: Create trajectory transformation utilities

**type:** auto

Create `ui/src/utils/trajectoryTransform.ts` with:
- `TrajectoryStep` interface (span, order, branch, timing)
- `ExecutionTrajectory` interface (steps, branches, duration)
- `transformToTrajectory(trace)` - Convert trace to trajectory
- `detectBranches(steps)` - Identify branch points
- `getExecutionPath(trajectory)` - Get main execution path

```typescript
interface TrajectoryStep {
  spanId: string;
  name: string;
  order: number;
  branch: 'main' | 'alternate' | 'error';
  startTime: number;
  duration: number;
  status: 'ok' | 'error';
  children: TrajectoryStep[];
}

interface ExecutionTrajectory {
  traceId: string;
  steps: TrajectoryStep[];
  branches: string[];
  totalDuration: number;
  mainPath: string[];
}
```

**done when:**
- File created with all interfaces
- Transformation correctly identifies execution order
- Branch detection works
