# Plan 06-01: Data Flow Diagram Generation from Traces

## Objective

Generate interactive data flow diagrams from trace span relationships, showing how data flows between agent actions, LLM calls, and tool invocations.

## Context

Building on Phase 3's trace visualization, this adds a diagram view that transforms span hierarchies into node-edge graphs. Uses Mermaid for rendering.

@CONTEXT.md

## Tasks

### Task 1: Create diagram transformation utilities

**type:** auto

Create `ui/src/utils/diagramTransform.ts` with:
- `SpanNode` interface (id, label, type, attributes)
- `DiagramEdge` interface (source, target, label, type)
- `transformToDiagram(trace)` - Convert trace spans to nodes/edges
- `detectFlowType(span)` - Determine if span is agent/llm/tool
- Layout algorithm for node positioning

```typescript
interface SpanNode {
  id: string;
  label: string;
  type: 'agent' | 'llm' | 'tool' | 'custom';
  x?: number;
  y?: number;
  attributes: Record<string, unknown>;
}

interface DiagramEdge {
  id: string;
  source: string;
  target: string;
  label?: string;
  type: 'parent-child' | 'data-flow' | 'call';
}

interface DiagramData {
  nodes: SpanNode[];
  edges: DiagramEdge[];
}
```

**done when:**
- File created with all interfaces and transformation function
- Function correctly maps span hierarchy to node-edge structure
- Unit tests pass
